# TASK SPEC — 100 Заключённых — 8-bit Browser Simulation

## Описание задачи

Браузерная симуляция классической задачи **«100 заключённых и 100 ящиков»**
в 8-битном NES-стиле. Один HTML-файл, чистый HTML/CSS/JS, без фреймворков.

Все надписи в интерфейсе — **на русском языке**.

---

## Правила задачи

- 100 заключённых, каждый под номером 1–100
- 100 ящиков, в каждом — номер одного из заключённых (случайная перестановка)
- Каждый заключённый может открыть не более 50 ящиков
- После открытия ящик остаётся открытым для следующих заключённых
- **Условие свободы:** все 100 заключённых должны найти свой номер
- Если хотя бы один не найдёт — казнь всех

---

## Состояния и переходы

### 1. Initial (стартовый экран)
- Сетка 10×10 закрытых ящиков с номерами снаружи (1–100)
- Инфо-панель пуста: Закл. «—», Нашли 0, Нет 0
- Кнопка СТАРТ активна, ПАУЗА/ШАГ заблокированы

### 2. Running (симуляция активна)
- **Для каждого заключённого (0..99):**
  - Определение следующего ящика для открытия (зависит от стратегии)
  - Анимация открытия ящика (показ внутреннего номера)
  - Подсветка текущего ящика пульсацией (`current-target`)
  - Если заключённый нашёл свой номер — зелёная маркировка этого ящика
  - Если не нашёл за 50 попыток или цикл — красная маркировка всех открытых им ящиков - в этом случае нет смысла продолжать - казнь всех
  - Переход к следующему заключённому
- Кнопки: ПАУЗА (остановка таймера), ШАГ (один шаг), СБРОС (сброс)

### 3. Paused
- Таймер остановлен
- Кнопка ПУСК возобновляет таймер
- ШАГ делает один шаг независимо от скорости

### 4. Done (итоговый экран)
- **УСПЕХ** (зелёный) — ≥51 нашли свой номер (математически эквивалентно «все 100»)
- **ПРОВАЛ** (красный) — ≤50 нашли
- Показывает число нашедших: «Нашли: X / 100 — Нужны ВСЕ 100»
- Кнопка OK закрывает экран

### 5. Серия (массовый режим)
- 1000 симуляций без анимации
- Прогресс-бар
- Счётчики: прогоны, ок, провал, процент успеха
- Гистограмма распределения найденных номеров
- Кнопка СТОП

## Стратегии

### Цепочка (оптимальная)
- Заключённый начинает с ящика, номер которого равен его собственному номеру
- Смотрит число внутри, идёт к ящику с этим номером
- Повторяет, пока не найдёт свой номер или не кончатся попытки
- Если цепочка зациклилась без находки — провал
- **Результат:** ~31% успеха (зависит от длины циклов в перестановке)

### Случайно (наивная)
- Заключённый выбирает 50 случайных (без повторений) ящиков
- Частичный Fisher-Yates: только первые 50 элементов перемешиваются
- **Результат:** ≈ (1/2)^100 ≈ 0% (практический ноль)

- `permutation: Array[100]` — перестановка чисел 1–100 (индекс = номер ящика, значение = номер внутри)
- `prisonersFound: boolean[100]` — какие заключённые нашли свой номер
- `currentChain: number[]` — индексы ящиков, открытых текущим заключённым

## Алгоритмы

### Перестановка (Fisher-Yates)
```
arr = [1, 2, ..., 100]
for i = 99 downto 1:
    j = floor(rng() * (i + 1))
    swap(arr[i], arr[j])
return arr
```

### Цепочка-стратегия (один заключённый)
```
boxIdx = prisonerNumber      // начать со своего номера
for attempt = 0 to 49:
    val = permutation[boxIdx]
    openBox(boxIdx)
    if val == prisonerNumber + 1:
        return SUCCESS
    nextIdx = val - 1
    if nextIdx уже открыт в этой попытке:    // цикл без находки
        return FAIL
    boxIdx = nextIdx
return FAIL                   // 50 попыток исчерпаны
```

### Случайная-стратегия (один заключённый)
```
indices = [0..99]
for i = 0 to 49:
    j = i + floor(random() * (100 - i))
    swap(indices[i], indices[j])
    boxIdx = indices[i]
    openBox(boxIdx)
    if permutation[boxIdx] == prisonerNumber + 1:
        return SUCCESS
return FAIL
```

## Критерии успеха

- Симуляция работает в браузере из одного файла
- Chain-стратегия показывает ~31% успеха при 1000+ прогонах
- Random-стратегия показывает ~0% успеха
- Анимация открытия ящиков с NES-эстетикой
- Пошаговый режим (ШАГ) для демонстрации
- Массовый режим с гистограммой
- Все надписи на русском языке
- Пиксельный шрифт Press Start 2P
